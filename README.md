# Thupten Wangpo

I build local-first developer tools, retrieval systems, and applied ML demos.

This profile points to the public work I want people to inspect first.

The common thread is operational control: keep private data local, make workflows inspectable, and ship tools that can be validated from the command line or a compact browser surface.

## Current Focus

- Fast local retrieval over private corpora and research archives.
- Agent workflow infrastructure for source capture, review loops, and tool boundaries.
- Applied ML demos that show the path from raw data to a working interface.
- Small utilities with clear privacy boundaries and reproducible validation commands.

## Working Style

- Prefer small tools with explicit inputs, outputs, and failure modes.
- Keep private material out of public repos by default.
- Write READMEs that say how to run, validate, and inspect the work.
- Treat demos as evidence: a repo should make the tradeoff or workflow visible.

## Public Boundary

Public repositories here are code, demos, and documentation. Private notes, research corpora, local indexes, credentials, and client-specific material stay outside the public GitHub surface.
